# CyberSentinel Demo Script (Presentation Runbook)

## Target demo length
8–12 minutes

## 1) Opening (45s)
- "CyberSentinel is our cyberattack detection and threat monitoring interface."
- "This frontend supports monitoring, incident awareness, notifications, and team accountability."

## 2) Dashboard walkthrough (2 min)
- Open `Dashboard`.
- Highlight:
  - threat stats cards
  - threat feed table
  - attack map panel
  - incidents overview
  - top attack types
  - system stats
- Mention: layout is responsive and optimized for operations screens.

## 3) Monitoring and incidents (1.5 min)
- Open `Threat Monitor` and `Attack Map`.
- Open `Incidents` and explain response visibility.
- Open `Defensive Console` to show operator tools concept.

## 4) Team and governance (1.5 min)
- Open `Users & Roles`.
- Show that each project workstream has an owner.
- Explain this supports accountability and handoff during operations.

## 5) Notifications flow (2.5 min)
- Open `Settings`:
  - show Twilio credentials section
  - add group email members
- Open `SMS / Twilio`:
  - send sample SMS (or dry-run)
  - click email button to launch prefilled email for group recipients
- Mention fallback email behavior (`mlestiwege@gmail.com`) if group list is empty.

## 6) Reporting (1 min)
- Open `Reports`.
- Show available report exports and contributors snapshot.

## 7) Testing and quality close (45s)
- Mention analyzer checks were run and pass in the current workspace.
- Mention Linux desktop run completed successfully in the latest session.
